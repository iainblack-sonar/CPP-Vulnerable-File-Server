# CPP-Vulnerable-File-Server

A minimal C++ project configured for SonarQube analysis demonstration.

## Project Structure

```
├── CMakeLists.txt          # Root CMake configuration
├── vcpkg.json              # vcpkg manifest for dependencies
├── sonar-project.properties # SonarQube configuration
├── include/
│   └── Version.h           # Version header file
├── src/
│   └── main.cpp            # Main application entry point
└── tests/
    ├── CMakeLists.txt      # Test CMake configuration
    └── test_main.cpp       # GoogleTest unit tests
```

## Dependencies

| Library | Purpose |
|---------|---------|
| OpenSSL | Encryption/TLS |
| SQLite3 | Database |
| GTest   | Unit testing |

## Prerequisites

- CMake 3.16+
- C++17 compatible compiler
- vcpkg package manager

## Building

### 1. Install vcpkg (if not installed)

```bash
git clone https://github.com/microsoft/vcpkg.git
cd vcpkg && ./bootstrap-vcpkg.sh
export VCPKG_ROOT=$(pwd)
```

### 2. Configure and Build

```bash
# Configure with vcpkg toolchain
cmake -B build -S . \
    -DCMAKE_TOOLCHAIN_FILE=$VCPKG_ROOT/scripts/buildsystems/vcpkg.cmake \
    -DCMAKE_BUILD_TYPE=Release

# Build
cmake --build build
```

### 3. Run

```bash
./build/CPP-Vulnerable-File-Server
```

### 4. Run Tests

```bash
cd build && ctest --output-on-failure
```

## SonarQube Analysis

1. Update `sonar-project.properties` with your project key
2. Ensure `build/compile_commands.json` exists (generated by CMake)
3. Run the SonarScanner:

```bash
sonar-scanner
```
